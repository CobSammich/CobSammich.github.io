Chapter 9: Error Handling

Rust has a number of features for handling situations when something may go wrong. Rust requires you to handle the possibility of an error before your code will compile. This makes your code more robust to any issues that could occur.

There are two major error categories: recoverable and unrecoverable errors. Recoverable errors allow for the program to continue to run by maybe making the user fix the name of an input file. Unrecoverable errors are often bugs, like trying to access an invalid array location.

Other programming languages handle errors with exceptions, but Rust doesn't have exceptions. Instead, it uses the type Result<T, E> for recoverable erros and the panic! macro to stop the program for unrecoverable errors.
