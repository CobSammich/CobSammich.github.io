Chapter 1: Getting Started

This chapter covers compilation and running of rust programs manually and using Cargo.

Writing and Running a Rust Program

The basic Hello, World! Program written in rust written in the file main.rs.

fn main() {
    println!("Hello, World!");
}

To compile the program:

$ rustc main.rs

Running the program should produce the following output:

$ ./main
Hello, World!

Anatomy of a Rust Program

The main function is essential in all rust programs. The main function is the first thing run a rust program. The main function can have parameters, but in this case it has none.

Why is there an exclamation mark after println? -- The ! defines a "macro". Chapter 19 explains macros in detail. For now just know that a macro behaves differently than a normal function.

Creating a Project with Cargo

You'll typically just use Cargo for compilation and running of our Rust programs. It handles those things.

Create a new project:

$ cargo new hello_cargo

This creates a hello_cargo/ directory with a Cargo.toml file and a src/ subdirectory with a Hello, World! program already written!

The Cargo.toml is shown below.

[package]
name = "hello_cargo"
version = "0.1.0"
edition = "2018"

[dependencies]

The dependencies section is where you would list all of your project's dependencies.

Building and Running a Cargo Project

To build a cargo project:

$ cargo build

This will create an executable in target/debug/hello_cargo. you can run this executable by entering the following command:

$ ./target/debug/hello_cargo
Hello, World!

You can compile and run the program with the single command:

$ cargo run

Note that Rust will only compile the project when it senses there is a change in the source files. Means that running/testing the program several times with cargo run is fine to do.

You can also check the program for errors without producing the binary by running cargo check.

Building for Release

You can build a release version of your project using cargo build --release. This will compile the project "with optimizations" making it faster to run, but also makes it take longer to compile. The release version is built in target/release/hello_cargo.

Summary

Let’s recap what we’ve learned so far about Cargo:

  • We can build a project using cargo build.
  • We can build and run a project in one step using cargo run.
  • We can build a project without producing a binary to check for errors using cargo check.
  • Instead of saving the result of the build in the same directory as our code, Cargo stores it in the target/debug directory.

Differentiating between debug and release:

  • Debug is best if you want to compile and run your code often
  • Release is best if you want the binary to be a production level optimized program.
