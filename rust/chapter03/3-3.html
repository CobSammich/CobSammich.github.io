Chapter 3.3: Functions

Functions are declared by the fn keyword. The main function is also declared by the fn keyword. Snake Case is the naming convention for functions and variables.

Below is an example of a program with a basic function in it. The main function calls the second function which prints Another function. after hello world.

fn main() {
    println!("Hello, world!");

    another_function();
}

fn another_function() {
    println!("Another function.");
}

Function Parameters

Functions can have parameters, which can be used within the function. To declare a parameter within a function definition, one would follow the syntax: (var_name: data_type). Below is an example of how a function would be passed in a 32 bit int and how that function would be called within main.

fn main() {
    another_function(5);
}

fn another_function(x: i32) {
    println!("The value of x is: {}", x);
}

Functions can obviously also have multiple parameters:

fn main() {
    print_labeled_measurement(5, 'h');
}

fn print_labeled_measurement(value: i32, unit_label: char) {
    println!("The measurement is: {}{}", value, unit_label);
}

Function Bodies Contain Statements and Expressions

Rust is made up of statements and expressions. Statements are instructions that perform some action and do not return a value. Expressions evaluate to a resulting value.

Creating a variable and assigning it a value is a statement. The code below shows this:

fn main() {
    let y = 6;
}

Statements do not return values. Thus you cannot assign a variable to another variable assignment statement. The code below attempts to do this and will result in an error:

fn main() {
    let x = (let y = 6);
}

The statement let y = 6 does not return a value, so that statement cannot be assigned to another variable. In the programming language C, you can do this by typing x = y = 6, but this is not valid in Rust.

Calling a function or macro is an expression and will evaluate to a value. Even if a function does not explicitly return something, the function call will evaluate to the value ().

In rust, however, new scopes are considered expressions. Thus, the expression below would evaluate to 4:

{
    let x = 3;
    x + 1
}

Functions with Return Values

A function is given a return value by typing an arrow (->) after the function definition. The return value is given by the final expression in the body of the function. You can also return early from a function by using the return keyword.

The simple code below shows how a function would return a value. Return statements don't need the return keyword if they don't end in a semicolon.

fn five() -> i32 {
    // these two lines are synonomous
    //return 5;
    5
}

fn main() {
    let x = five();
    println!("The value of x is: {}", x);
}
