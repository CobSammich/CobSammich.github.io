Enumerated Data Types

You can define Enums using the following syntax:

enum class Day {
    Monday,
    Tuesday,
    Wednesday,
    Thursday,
    Friday,
    Saturday,
    Sunday
};

Each enumerator will be automatically defined to have a fixed integer value of type int by default. The first nmame in the list, in our case Monday, will have the value 0, Tuesday will be 1, and so on.

You can define today as a variable of the enumaration type Day with the following syntax:

Day today {Day::Tuesday}

You can have enums that include literals. For example:

enum Class Punctuation : char {Comma = ',', Exclamation = '!', Question = '?'};

Aliases for Data Types

You can use the using keyword to specify a type alias for a data type. For example, you can define the type alias BigOnes as being equivalent to the standard type unsigned long long:

using BigOnes = unsigned long long;
BigOnes value; // initialize value of unsigned long long

Another way to create a type alias is using the following older syntax:

typedef unsigned long long BigOnes;
