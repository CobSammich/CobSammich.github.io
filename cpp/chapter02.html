Variables, Data, and Data Types

Defining Integer Variables

One can define integers using one of three syntaxes:

int apple_count = 15;
int apple_count {15};
int apple_count(15);

Integers can, well only be integers, but they can be defined as a float in some intialization syntaxes. The compiler handles the different assignment operations differently:

int banana_count(7.5);         // May compile without warning
int coconut_count = 5.3;       // May compile without warning
int papaya_count{0.3};         // At least a compiler warning, often an error

In the cases where the compiler does not complain, the floating point values get rounded down to the nearest integer value. banana_count will be $7$, and coconut_count will be $5$.

You can also define and initialize multiple variables on a single line:

int foot_count {2}, toe_count {10}, head_count {1};

Although this is valid to the compiler, it is considered best practice to define each variable on it's own line as it is considered more readable.

Zero Initialization

You can define a variable to be zero in one of the following two ways:

int counter = 0;
int counter;

The second statement still initializes the variable counter to be 0.

Defining Variables with Fixed Values

To define constant values, that is values that cannot be changed throughout the program, we can use the const keyword:

const unsigned toe_count = 10;  // An unsigned integer with fixed value 10

The const keyword tells the compiler that this variable must not be modified. Thus, any attempt to modify it in the program will result in a compilation error.

Integer Literals

Constant values of any type, such as 42, 2.71828, 'Z', or "Mark Twain" are referred to as literals. Specifically they are integer literals, floating point literals, character literals, and string literals.

Literals can be defined in some of the following ways:

  • -123L - type long
  • +123 - type int
  • 123 - type int
  • 22333 - type int
  • 98U - type unsigned int
  • -1234LL - type long long
  • 12345ULL - type unsigned long long

To make large numeric values readable, you can use the single quote character:

const long long distance AU_METERS = 149'597'870'700 // Astronomical unit is 150 billion meters

Hexadecimal Literals

You can write integer literals as Hexadecimal values. A hexadecimal literal is prefixed with a 0x

Each hexadecimal digit corresponds to 4 buts, so it's easy to express a pattern of bits as a hexadecimal literal. An RGB value of a pixel color is often expressed as three bytes packed into a 32-bit word. The color white can be specified as 0xFFFFFF because each intensity of each component have the same max value of 255 or 0xFF in hexadecimal. Another example is the color red, which would be defined as 0xff0000.

Octal Literals

You can also write literals as octal values -- that is, using base 8 instead of hexadecimal which uses base 16. An octal is prefixed by a leading 0.

The octal value 0657 has the decimal literal 431. To convert from octal to decimal, you would do the following computation: $$ 6\times8^2 + 5\times8^1 + 7\times8^0 = 6 \times 64 + 5 \times 8 + 7 \times 1 = 431 $$

The sizeof Operator

You can use the sizeof operator to obtain the number of bytes occupied by a type, by a variable, or by the result of an expression. The code below shows three examples of using sizeof on a variable, a type, and the result of an operation.

int height {74};
std::cout << "The height variable occupies " << sizeof height << " bytes." << std::endl;
std::cout << "Type \"long long\" occupies " << sizeof(long long) << " bytes." << std::endl;
std::cout << "The result of the expression height * height/2 occupies "
          << sizeof(height * height/2) << " bytes." << std::endl;

Explicit Type Conversion

To explicitly convert a value of an expression to a given type, you can write the following static_cast<type_to_convert_to>(expression). Below is a practical example:

int y {};
double z {5.0};
y = static_cast<int>(z);

Old-Style Casts

In older versions of C++, explicit type casts were defined in the following manner: (type_to_convert_to)expression

And here is a practical example:

inches  = (unsigned int)(length * feet_per_yard * inches_per_foot) % inches_per_foot;

The auto Keyword

You can use the auto keyword to indicate that the compiler should deduce the type.

auto m = 10;        // int
auto n = 200UL;     // unsigned long
auto pi = 3.14159;  // double
